  • No Ruby installed on the target. No bundle. Just a file.

What is Spinel?

A whole-program AOT Ruby-to-C compiler

Reads your Ruby, emits C, links a native executable.

Buys you

Instant startup, trivial deploys, one self-contained binary.

Costs you

No eval, no runtime metaprogramming, a curated stdlib.

How we'll tour this

5 apps, each built and verified

  • FyelSrvr, static file server (sockets)
  • todo_cli, CLI task manager (CLI flags + JSON)

  • token_api, JSON API with signed tokens (FFI)

Every one compiles, runs, and also runs under plain ruby.

Technique · one Ruby surface, two runtimes

socket_shim.rb

# socket_shim.rb
if defined?(RUBY_ENGINE)
  require "socket"          # CRuby: the real TCPServer
end
if defined?(TCPServer)
  # already have it
else
  module SpinelSocketNative # Spinel: FFI over sp_net
    ffi_func :sp_net_listen, [:int, :int], :int
    ffi_func :sp_net_accept, [:int], :int
  end
end
  • Same TCPServer.open / recv / write API on both.

Deep dive · how did we KNOW sockets needed FFI?

The gap-diagnosis decision tree

1. Try plain Ruby.        require "socket"
   $ spinel -E -e 'require "socket"; puts 1'
   spinel: cannot load such file -- socket      # gone

2. Try the require gate.  SPINEL_REQUIRE_GATE=1
   $ SPINEL_REQUIRE_GATE=1 spinel -E -e 'require "socket"'
   cannot load such file -- socket   # not a gated pkg either
  • No Ruby surface, no package, but a TCP server is core OS work, so the capability almost certainly exists in the runtime.
  • The question flips: not “is it in Ruby?” but “is there a native sp_* symbol I can bind?”

Deep dive · find the native symbol, then bind it

grep the runtime's lib/, read the C signature

3. Grep the Spinel runtime for the capability.
   $ grep -rn "sp_net" spinel/lib/*.h
   lib/sp_net.h: int sp_net_listen(int port, int reuseport);
   lib/sp_net.h: int sp_net_accept(int fd);
   lib/sp_net.h: char *sp_net_recv_some(int fd, int max);

4. Translate the C signature into an ffi_func line:
   int  sp_net_listen(int, int)  →  [:int, :int], :int
   char*sp_net_recv_some(int,int)→  [:int, :int], :binstr
  • Types: :int, :str (NUL-terminated), :binstr (length from sp_net_bin_len), :void.
  • Rule of thumb: a header in lib/sp_*.h = a bindable symbol. Most of lib/ backs Ruby methods, only reach for the ones with no Ruby surface (net, crypto).

Deep dive · the module = the binding table

module + ffi_func, then a Ruby wrapper

module SpinelSocketNative        # the binding table
  ffi_func :sp_net_listen,   [:int, :int], :int
  ffi_func :sp_net_recv_some,[:int, :int], :binstr
end

# call it: Module.symbol(args)
fd = SpinelSocketNative.sp_net_listen(port, 1)
  • ffi_func :name, [arg_types], :ret, module-level, one line per symbol. No headers, no attach_function boilerplate.
  • Then wrap the raw fds in familiar Ruby classes (TCPServer/TCPSocket) so callers never see the FFI.
  • Same recipe reused for crypto next, sp_crypto_* in lib/sp_crypto.h.

Deep dive · bring your OWN C

Write C, bind it, link it with --link

// native/my_math.c
int my_add(int a, int b) {
  return a + b;
}
# app.rb
module MyNative
  ffi_func :my_add, [:int, :int], :int
end
puts MyNative.my_add(2, 3)   # => 5
# compile + link your .c straight in:
$ spinel --link native/my_math.c \
    source/app.rb -o bin/app
$ ./bin/app
5

# probe form: -E runs it, keep --link
$ spinel --link native/my_math.c -E app.rb
  • --link ARG is repeatable: a .c, .o, .a, or -lNAME.
  • ffi_func emits a plain extern; the linker finds your symbol. No name mangling.

Deep dive · the custom-C ABI

Signatures, and returning strings safely

// :str is a borrowed const char* (NUL-terminated).
// For bytes, return :binstr and set the length global:
extern int sp_net_bin_len;          // runtime-provided
static char buf[65536];             // static, not stack
const char *my_bytes(const char *in) {
  int n = fill(buf, in);            // ...produce n bytes
  sp_net_bin_len = n;               // MUST set for :binstr
  return buf;
}
  • Types map 1:1: :int→int, :str/:binstr→const char*, :ptr→void*, :void→void.
  • Returned buffers are borrowed & static, same rule as the runtime: copy with "" + s to keep it, Mutex it under threads.
  • Need custom CFLAGS/static linking? spinel app.rb -c to emit C, then drive cc yourself.

Technique · path-traversal defense

sanitize_path

segments = clean.split("/")
safe_segments = []
segments.each do |segment|
  next if segment == "" || segment == "."
  if segment == ".."
    safe_segments.pop     # can never escape root
  else
    safe_segments.push(segment)
  end
end
"/" + safe_segments.join("/")

HTTP/1.1 404 Not Found

$ ./todo_cli done 1

Technique · JSON round-trip persistence

Durable state in one file

require "json"

File.write(path, JSON.pretty_generate(tasks))
tasks = JSON.parse(File.read(path))   # string keys on load
  • JSON.pretty_generate, JSON.parse, File.write/read.
  • Flags parsed with a tiny ARGV loop, Spinel's optparse subset captures boolean flags only, not option values.

Technique · stdlib via the require gate

SPINEL_REQUIRE_GATE=1

$ SPINEL_REQUIRE_GATE=1 spinel source/todo_cli.rb -o bin/todo_cli
  • Gated packages: json, optparse, set, strscan, stringio, base64, digest.
  • Not available: date, net/*, securerandom; erb is a stub.
  • Probe first: a 1-line -e check before you commit to an API.

App 4 / 5

token_api

A JSON API with HMAC-signed bearer tokens

POST /login  {"user":"matz"}   → {"token":"matz.6Q_..."}
GET  /notes  (Bearer token)    → 200 [ ...notes... ]
GET  /notes  (no/bad token)    → 401 unauthorized

Hand-rolled routing on the socket shim. No framework.

Technique · the same FFI recipe, second time

Call the C runtime directly

# require "openssl"? gone. grep lib/sp_crypto.h → found:
module SpinelToken
  ffi_func :sp_crypto_hmac_sha256_b64url, [:str, :str], :str
  ffi_func :sp_crypto_random_b64url, [:int], :str
end
  • Exact steps as sockets: probe → gate → grep lib/ → declare. Crypto lives in lib/sp_crypto.h.
  • Under CRuby the same file falls back to OpenSSL::HMAC.
  • Byte-identical tokens across both runtimes.

War story · the static-C-buffer gotcha

Copy before the next call

# sp_crypto_* return a shared static buffer that the
# NEXT crypto call overwrites. Copy it out first:
def self.hmac_b64url(key, msg)
  "" + SpinelToken.sp_crypto_hmac_sha256_b64url(key, msg)
end
  • "" + value materializes a fresh, safe Ruby String.
  • Same rule bit us again in parallel_digest, next up.
App 5 / 5

parallel_digest

A worker pool: Thread + Queue + Mutex

$ ./parallel_digest source -w 4
45f1b6f59acf  source/todo_cli.rb
ac581dce3fc1  source/token_api.rb
hashed 11 files with 4 workers

Technique · the shutdown handshake

Queue + poison pills

files.each { |f| queue << f }
workers.times { queue << nil }   # one pill per worker

workers.times do
  threads << Thread.new { worker_loop(queue, results, mutex) }
end
threads.each { |t| t.join }
  • Worker: item = queue.pop; break if item.nil?
  • Queue#close → pop returns nil is the other idiom.

Technique · Ruby threads with NO GVL

Real parallelism → Mutex is mandatory

mutex.synchronize do
  results << "#{path}\t#{digest}"   # shared state
end
  • Spinel schedules Threads M:N onto OS workers, the JRuby/TruffleRuby model, not CRuby's GVL.
  • So unguarded Array#<< from two threads is a real data race.
  • synchronize is compiler-inlined with full ensure semantics.

Live knob · and an honest caveat

SPINEL_WORKERS + a thread-safety limit

$ SPINEL_WORKERS=1 ./parallel_digest source -w 8
$ ./parallel_digest source -w 8       # uses all cores
  • Pure CPU work parallelizes: ~3.5× on a 4-thread probe.
  • But File.read / Digest use shared static buffers, not thread-safe, so we serialize the read+hash in the Mutex.
  • The lesson: no GVL means shared/static libs need a critical section.

Honest limitations

What didn't fit

  • No eval / runtime metaprogramming; reflection mostly gone.
  • No date, no net/*; erb is a stub; no Time.parse.
  • Threading: data races are observable (no GVL); Thread#raise/#kill on the main thread is a no-op.
  • Some stdlib helpers (File/Digest) aren't thread-safe yet.

Patterns that carried across all 5 apps

How to work with Spinel

  • Probe first: spinel -E -e '...' before writing the app, -e is inline Ruby source, -E compiles & runs it on the spot (no file, no leftover binary).
  • Require gate: SPINEL_REQUIRE_GATE=1 for stdlib packages.
  • FFI last: use Ruby → use require → use require_relative → only then ffi_func for native symbols.
  • Copy static buffers and normalize strings with "" + s.
